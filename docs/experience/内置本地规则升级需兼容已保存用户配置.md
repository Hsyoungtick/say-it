# 内置本地规则升级需兼容已保存用户配置

## 触发条件

- 修改 `ui/src/features/dictation/localRulesEngine.ts` 里的内置规则默认值、名称、说明或执行语义。
- 这些规则会被持久化到 `localStorage` 的 `sayItDictPrefs.localRules`。

## 正确做法

- 不能只改 `presets()` 默认值；老用户读取到的仍是本地已保存的旧规则。
- 需要在 `mergeLocalRules()` 中补升级逻辑：
  - 只对“仍等于历史默认快照”的内置规则自动升级。
  - 保留用户改过的规则内容、顺序和启停状态，不强行覆盖。
- 如果某条内置规则已经不再适合只靠正则表达，可保留它在规则表中的可见形态，再在执行阶段按 `id` + 默认快照做特殊处理，避免误伤用户自定义改写。

## 这次场景

- `dedupe-punct` 从“合并连续重复标点”升级为“合并连续标点”。
- 为了让旧配置自动拿到新行为，同时不覆盖用户自己改写过的规则，需要同时修改：
  - `presets()` 里的默认规则定义；
  - `mergeLocalRules()` 里的兼容升级判断；
  - `applyRulesPure()` 里的内置特殊执行逻辑。

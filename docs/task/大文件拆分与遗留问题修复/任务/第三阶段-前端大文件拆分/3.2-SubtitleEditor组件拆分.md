# 3.2 SubtitleEditor 组件拆分

## 任务状态

- 状态：待开始
- 所属阶段：第三阶段-前端大文件拆分
- 前置任务：无
- 关联任务：无
- 是否可以并行：可与 1.x、2.x、3.1 并行

## 任务目标

把 1067 行的 `ui/src/components/transcription/SubtitleEditor.tsx` 拆分为「常量与工具 / 波形绘制 / 小型子组件 / 主编辑器」多个文件，主组件文件显著瘦身；组件对外 props、交互行为与视觉完全不变，消费方 import 路径不变。

## 当前情况

- 单文件结构（行号为 2026-07-06 时点）：
  - **常量**（11~33）：像素密度、吸附距离、缩放档位、时间线/波形几何尺寸、播放头校正参数等约 20 个。
  - **类型与工具**（35~194）：`DragMode`/`DragState`/`PanState`/`WaveformColumn`/`CueNeighbors`、`clamp`/`formatZoom`/`isTypingTarget`/`isInteractiveTarget`/`yieldToMain`/`joinTexts`。
  - **波形绘制**（137~187）：`drawWaveformCanvas`（canvas 渲染，纯函数）。
  - **子组件**（196~305）：`TimeInput`、`CueTextarea`、`TimeControl`（均为独立受控组件）。
  - **主组件**（306~1067，约 760 行）：`SubtitleEditor`——时间线拖拽（move/left/right 三种模式）、平移、吸附、缩放、波形交互、播放头跟随与校正、cue 增删改、快捷键，大量 useRef/useEffect。
- 消费方仅两处：`ui/src/views/TranscriptionView.tsx:10`、`ui/src/views/TranscriptAlignPanel.tsx:6`，均为 `import { SubtitleEditor } from "@/components/transcription/SubtitleEditor";`。
- 本组件是重鼠标交互组件（拖拽/吸附/缩放），按 CLAUDE.md 约定，改后必须交由用户实测，不得自行模拟验证。

## 实施方案

同目录建 `subtitle-editor/` 子目录放辅助文件，`SubtitleEditor.tsx` 保留在原路径作为主组件（消费方零改动，也符合「组件按文件单独导入、无 barrel」的仓库约定）：

```text
ui/src/components/transcription/
├── SubtitleEditor.tsx            # 仅主组件（拖拽/缩放/播放头等核心交互逻辑）
└── subtitle-editor/
    ├── constants.ts              # 全部布局/交互常量
    ├── utils.ts                  # clamp/formatZoom/isTypingTarget/isInteractiveTarget/yieldToMain/joinTexts + DragMode 等共享类型
    ├── waveform.ts               # drawWaveformCanvas + WaveformColumn
    ├── TimeInput.tsx              # 时间输入子组件
    ├── CueTextarea.tsx            # 字幕文本子组件
    └── TimeControl.tsx            # 时间调节子组件
```

要点：

- 主组件内部约 760 行的交互逻辑**本任务不强行继续拆**（拖拽状态机与 refs 高度耦合，硬拆自定义 hook 风险大于收益）；仅当执行时发现某段逻辑（如波形数据加载）与拖拽状态机零耦合，才可提为局部 hook，并在执行记录说明。
- 全部搬运不改逻辑；类型/常量/工具的可见性按「只导出被跨文件使用的项」收敛。

## 执行步骤

1. 建 `subtitle-editor/` 目录，依次搬 `constants.ts` → `utils.ts` → `waveform.ts`，每步 `npx tsc --noEmit`。
2. 搬三个子组件（`TimeInput`/`CueTextarea`/`TimeControl`）为独立 `.tsx`。
3. `SubtitleEditor.tsx` 更新 import；确认两个消费方视图零改动。
4. `npx tsc --noEmit`、`npm run ui:build` 通过。
5. 向用户列出实测清单（见验收标准），等待确认。

## 任务目标

## 当前情况

- 单文件实现了完整的「ASR 词序列 ↔ 文稿行」对齐流水线，唯一消费方是 `commands/transcription.rs:12`（`use crate::text_align::{align_script, AlignOutput, AlignWord};`），`main.rs:12` 声明 `mod text_align;`。
- 内部结构（行号为 2026-07-06 时点）分四段清晰的子职责：
  - **分词与字符归一**：`tokenize_text`（461）、`tokenize_with_spans`（467）、`flush_latin_spans`（489）、`canonical_char`（495）、`is_cjk`（519）、`build_asr_tokens`（429）、`AsrToken`（128）。
  - **对齐核心**：`intern_ids`（533）、`align_tokens`（550）、`find_anchors`（599）、`longest_increasing_by_a`（641）、`nw_full`（674）、`nw_banded`（778）、`best3`（50）、`TokenLink`（119）。
  - **行/段构建与后处理**：`build_aligned_lines`（170）、`build_optimized_segments`（238）、`reclassify_short_bad_runs`（294）、`reclassify_hitless_good_runs`（317）、`hit_asr_index`（342）、`build_keep_segment`（352）、`collect_orphan_segments`（403）、`fill_missing`（925）、`post_process`（966）。
  - **测试**：1000 行起约 260 行，覆盖中文数字、锚点、带状回退等回归场景。
- 该文件是纯算法、职责边界清楚，拆分收益主要是可读性与可导航性，风险是搬运时误改算法细节——必须逐字搬运。

## 实施方案

`text_align.rs` → `text_align/` 目录：

```text
src-tauri/src/text_align/
├── mod.rs       # 公共类型（AlignWord/AlignedLine/OptimizedSegment/AlignOutput）+ align_script 编排函数 + pub use
├── tokenize.rs  # 分词、字符归一、AsrToken 构建
├── align.rs     # TokenLink、intern、锚点、LIS、nw_full/nw_banded、best3
├── segments.rs  # 行/段构建、重分类、孤儿段、fill_missing、post_process
└── tests.rs     # 原 #[cfg(test)] 全部测试（#[cfg(test)] mod tests; 方式挂载）
```

- 跨子模块可见性用 `pub(super)`；对外 pub 面不变，`main.rs` 与 `commands/transcription.rs` 零改动。
- 测试统一放 `tests.rs`（它们都走 `align_script` 顶层入口，不依赖子模块私有函数；执行时若发现有测试直接调私有函数，则该测试留在对应子模块内）。

## 执行步骤

1. 建目录，把原文件内容整体移入 `mod.rs`，`cargo test` 确认基线。
2. 按「分词 → 对齐核心 → 行/段构建」顺序逐块搬到子文件，每块搬完 `cargo check`。

4. 收敛可见性，消除未使用警告。
5. `git diff` 复查：`main.rs`、`commands/transcription.rs` 无改动。

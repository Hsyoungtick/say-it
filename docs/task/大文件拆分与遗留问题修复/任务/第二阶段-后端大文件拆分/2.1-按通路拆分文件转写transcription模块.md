## 任务目标

把 944 行的 `src-tauri/src/providers/alibabacloud/transcription.rs` 拆为 `transcription/` 模块目录，按「共享类型 / 三条通路 / 音频编码 / HTTP 工具」分文件，每个文件职责单一；对外 pub 面经 `mod.rs` re-export 保持完全不变，行为零变化。

## 当前情况

单文件混装了以下职责（行号为 2026-07-06 时点）：

- **共享类型与参数**（约 14~260 行）：`TranscriptionParams`（含 `Default` 与请求参数组装）、`TranscriptionTaskStatus/TaskResult`、`TranscriptionResult/Transcript/Sentence/Word`、内部反序列化结构（`SubmitResponse`、`TaskResponse`、`RawTranscriptionResult` 等）、`TranscriptionModelFamily` 枚举。
- **异步 OSS 通路**：`submit_transcription_task`（262）、`query_transcription_task`（447）、`fetch_transcription_result`（476）、`transcription_input_value`（523）。
- **同步短音频通路（两条混在 `recognize_short_audio`，305 起，约 140 行）**：内部按 family 分支走 Fun-ASR-Flash SSE 或 Qwen 非流式。
- **Fun-ASR-Flash SSE 解析与回声去重**：`parse_fun_asr_flash_sse`（541）、`last_sentence_still_finalizing`（615）、`next_sentence_is_stale_echo`（635）、`is_duplicate_sentence_text`（646）、`extract_sse_error_message`（658）。
- **Qwen 短音频结果解析**：`parse_short_audio_result`（665）、`short_audio_content_text`（695）。
- **音频编码（与转写协议无关）**：`build_opus_data_uri`（713）、`build_mp3_data_uri`（727）、`encode_mp3_mono`（737）。
- **HTTP 工具**：`read_json_response`（775）、`extract_error_message`（792）、`format_task_error`（803）、`truncate`（815）。
- **测试**：825 行起，含 SSE 去重回归测试与 Opus/MP3 编码测试。

对外消费方：`alibabacloud/mod.rs` re-export 了 `fetch_transcription_result, query_transcription_task, recognize_short_audio, submit_transcription_task, uses_async_transcription_task, TranscriptionParams, TranscriptionTaskStatus`；命令层 `commands/transcription.rs` 经 prelude 使用这些项。模型→通路判断已收敛到 `providers/registry.rs`（`file_transcription_route`），本文件的 `transcription_model_family` 只是在 AsyncOss 内进一步细分。

## 实施方案

`transcription.rs` → `transcription/` 目录，纯搬运：

```text
providers/alibabacloud/transcription/
├── mod.rs          # 模块声明 + pub use（对外 pub 面与现状完全一致）+ TranscriptionModelFamily + transcription_model_family + uses_async_transcription_task + default_transcription_model
├── types.rs        # TranscriptionParams / TaskStatus / Result 系列公共类型 + 内部反序列化结构
├── async_oss.rs    # submit / query / fetch + transcription_input_value
├── sync_flash.rs   # recognize_short_audio 的 Fun-ASR-Flash SSE 分支 + parse_fun_asr_flash_sse + 去重三函数 + extract_sse_error_message + 对应测试
├── sync_qwen.rs    # Qwen 短音频分支 + parse_short_audio_result + short_audio_content_text
├── media_encode.rs # build_opus_data_uri / build_mp3_data_uri / encode_mp3_mono + 编码测试
└── http.rs         # read_json_response / extract_error_message / format_task_error / truncate
```

要点：

- `recognize_short_audio` 保留为对外入口（放 `mod.rs` 或按分支归属拆到两个 sync 文件后在 mod.rs 组装，执行时选改动更小者），内部 family 分支逻辑逐字保留。
- 函数体不做任何逻辑修改；跨文件调用把原私有函数升为 `pub(super)`，不升为 `pub`。
- `alibabacloud/mod.rs` 的 `pub use transcription::{...}` 列表不变。

# 3.2 升级音频调试与本地规则面板

## 任务状态

- 状态：已完成
- 所属阶段：第三阶段-设置与复杂控件统一
- 前置任务：1.2
- 关联任务：2.1、3.1、4.1、4.2
- 是否可以并行：可以。可与 2.1、2.2、3.1 在 1.2 完成后并行，但需避免同时修改 `Slider`、`Input`、`Button` 等基础组件。

## 任务目标

升级音频调试和本地文本处理规则等复杂高密度界面，使其在保留专业信息密度的同时，与新 UI 系统统一，并确保调试、规则编辑、预览和状态反馈清晰可用。

## 当前情况

- `ui/src/views/AudioView.tsx` 管理录音试听、A/B 播放、实时电平、波形 canvas、响度归一化、RNNoise 降噪、EQ 参数。
- `ui/src/views/LocalRulesPanel.tsx` 管理本地查找替换、正则规则、启用状态、规则排序、预览输入输出和错误提示。
- `ui/src/views/FunAsrHotwordsPanel.tsx` 管理热词列表，通常嵌入供应商配置中。
- `ui/src/components/ui/LogPanel.tsx` 用于调试日志显示。
- 这些界面控件多、密度高，当前样式更偏局部实现，输入框、列表、按钮、错误状态和 canvas 容器需要统一。

## 实施方案

以“桌面端高密度工具面板”为目标，不扩大功能范围。音频调试保留仪表和波形 canvas；本地规则保留当前规则数据结构与 worker 预览；热词面板只在样式需要时局部统一。

# 4.1 窗口尺寸与构建验证

## 任务状态

- 状态：已完成
- 所属阶段：第四阶段-适配验证与收尾
- 前置任务：2.1、2.2、3.1、3.2
- 关联任务：4.2
- 是否可以并行：否。需要主要 UI 改造完成后执行。

## 任务目标

对改造后的主窗口 UI 进行构建、类型、窗口尺寸和基础视觉验证，确认在常见桌面窗口尺寸下没有明显重叠、截断、横向溢出或构建失败。

## 当前情况

- 当前项目没有配置 `lint`、`format`、`test` 脚本。
- 可用检查命令包括 `npx tsc --noEmit`、`npm run ui:build`，完整桌面开发可用 `npm run tauri:dev`。
- Vite 开发端口固定为 `5155`，且 `strictPort: true`。
- 主窗口入口为 `ui/index.html`，悬浮窗入口为 `ui/indicator.html`，构建会同时处理两个入口。
- 用户要求至少考虑 1366×768、1440×900、1920×1080、Windows 125% 缩放、Windows 150% 缩放。

## 实施方案

先执行类型与构建验证，再启动本地前端或 Tauri 开发环境检查主要页面布局。能自动验证的内容由执行者完成；涉及鼠标交互和真实桌面行为的内容只记录步骤，交给 4.2 和用户人工验证。

## 执行步骤

1. 检查 `git status --short`，确认当前任务只包含 UI 改造相关改动。
2. 运行 `npx tsc --noEmit`，记录结果。
3. 运行 `npm run ui:build`，记录结果。
4. 如需要运行页面，先确认端口 5155 未被占用，再运行 `npm run ui:dev` 或 `npm run tauri:dev`。
5. 检查主窗口主要页面在 1366×768、1440×900、1920×1080 下的布局：侧栏、标题栏、页头、tabs、表单、上传区、结果区、设置页、滚动容器。
6. 检查较低高度窗口下是否可滚动，主内容是否无横向溢出。
7. 记录 Windows 125% 和 150% 缩放的验证结果；如当前环境无法验证，明确记录“未验证”和原因。
8. 修复只属于本任务范围的布局问题；若发现业务或交互问题，记录到 4.2 或对应任务，不扩大范围。
9. 更新本任务执行记录和 `00-任务总览.md`。

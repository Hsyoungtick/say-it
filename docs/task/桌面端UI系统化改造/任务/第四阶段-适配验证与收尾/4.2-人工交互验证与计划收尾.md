# 4.2 人工交互验证与计划收尾

## 任务状态

- 关联任务：全部任务
- 是否可以并行：否。需要构建和基础尺寸验证完成后执行。

## 任务目标

整理并执行需要用户参与的桌面端交互验证，确认 UI 改造没有破坏鼠标、键盘、拖放、悬浮窗和文件选择等真实桌面流程，并完成任务计划收尾。

## 当前情况

- 用户明确要求涉及鼠标操作、拖拽、点击、悬浮、画布交互等需要实际操作 UI 的场景，不由 AI 自行上手验证，而是写清操作步骤和验证点交给用户测试。
- 需要人工验证的关键交互包括：侧栏切换、tabs、Select 展开和搜索、快捷键捕获、文件拖放、字幕预览、滑块、颜色选择、音频录放、规则编辑、折叠面板。
- README 截图是否同步替换仍为待确认。

## 实施方案

基于所有已完成任务的执行记录，整理一份人工验证清单。执行者可以进行非鼠标类命令检查，但真实交互由用户按步骤完成。用户反馈问题后，按影响范围决定修复当前任务、重开相关任务或记录待确认事项。

## 执行步骤

1. 阅读 2.1、2.2、3.1、3.2、4.1 的执行记录，汇总所有需要人工验证的交互。
2. 在本任务执行记录中写清用户验证步骤，至少覆盖：
   - 侧栏视图切换和页面滚动。
   - 语音输入快捷键修改、取消、注入方式选择。
   - 实时字幕开始/停止、预览开关、滑块和颜色选择。
   - 录音识别文件选择、拖放、重新选择、结果视图切换和导出。
   - 设置页 API Key 显示/隐藏、折叠面板、启动设置、提示音选择。
   - 音频调试录音/播放、规则新增/删除/排序和试运行。
3. 用户完成验证后，根据反馈修复本次 UI 改造引入的问题；若问题属于新需求或范围外，记录为待确认。
4. 根据实际情况确认是否更新 `docs/images` 截图；未确认则保持不变并记录。
5. 检查所有任务文件状态、执行记录、重要记录和总览是否同步。
6. 最终检查 `git status --short` 和 `git diff`，确认只包含预期改动。
7. 为收尾结果创建独立提交。

## 人工交互验证清单（请在 `npm run tauri:dev` 下按项验证）

> 说明：以下标注 ★ 的项目为涉及鼠标/键盘/拖放/悬浮窗/文件选择/DPI 缩放的真实桌面交互，需由用户实际操作确认；未标注项可在目视时一并核对。发现问题请在“执行记录-用户反馈”中记录。

### A. 应用壳层与导航
- ★ 侧栏在「语音输入 / 实时字幕 / 录音识别 / 设置」间切换，选中态为实心强调色高亮，内容区正确切换。
- ★ 各页面内容超出高度时，主内容区可纵向滚动，标题栏与侧栏固定不动，无页面级横向滚动条。
- ★ 拖动标题栏可移动窗口；右上角最小化/关闭按钮可点击生效（无边框窗口）。
- 页头（标题+描述）、分组标题、卡片间距在各页一致。

### B. 语音输入页
- ★ Tabs「通用设置 / 文本处理 / 调试」切换正常。
- ★ 通用设置：修改语音输入快捷键（捕获按键组合）、取消捕获、注入方式选择，保存后生效。
- ★ 调试页：「手动开始 / 停止并注入」按钮切换录音；「清空日志」可用；「输出调试日志」勾选框可切换；最近识别只读文本框与日志面板显示正常。
- 状态文本颜色随状态（普通/成功 ok/错误 err）正确变化。

### C. 实时字幕页
- ★ 开始 / 停止字幕捕获按钮生效。
- ★ 字幕预览开关切换；滑块（字号/位置等）拖动生效；颜色选择器可选色并即时预览。
- ★ 两列表单在窄窗下退化为单列，无重叠/截断。

### D. 录音识别页 + 文稿对齐
- ★ 点击选择音频文件；★ 从资源管理器拖放文件到上传区；★ 重新选择文件。
- ★ 识别完成后结果视图切换（如逐句/纯文本）；★ 导出/保存文本文件。
- ★ 文稿对齐结果表（时间/文本多列）在窄窗下退化为单列堆叠，无横向溢出。
- 上传区、状态条、结果区在识别前/中/后三态显示正确。

### E. 设置页
- ★ 供应商配置：切换供应商、编辑 API Key、显示/隐藏 API Key、保存（含自动保存）后再次进入保持。
- ★ 折叠/展开各配置面板。
- ★ 启动设置（开机自启等）开关切换。
- ★ 麦克风与提示音：选择麦克风设备、选择/试听提示音。
- ★ 外观：暗色/亮色切换、强调色选择即时生效且不破版（亮色为“可用不破版”标准，见记录 004）。

### F. 音频调试 + 本地规则（文本处理）
- ★ 音频调试：录音 / 播放回放；静音测试运行。
- ★ 本地规则：新增 / 删除 / 排序规则；规则「启用」开关（Switch）切换；试运行预览输出。
- ★ 热词：新增 / 保存 / 清空热词后同步生效。
- ★ 日志面板显示与滚动正常。

### G. 悬浮指示窗（indicator）
- ★ 触发语音输入/字幕时悬浮窗出现，位置/文本/状态显示正确，不受主窗口 UI 改造影响。

### H. 窗口尺寸与 DPI（承接 4.1）
- ★ 分别在 1366×768、1440×900、1920×1080 窗口尺寸下目视核对：无重叠、截断、横向溢出。
- ★ Windows 显示缩放 125% 与 150% 下目视核对：文字/控件清晰，布局不错位。

# 1.1 梳理视觉规范并建立设计令牌

## 任务状态

- 状态：已完成
- 所属阶段：第一阶段-设计系统与应用壳层
- 前置任务：无
- 关联任务：1.2、2.1、2.2、3.1、3.2、4.1
- 是否可以并行：否。本任务是后续 UI 改造基础。

## 任务目标

建立当前项目可持续使用的 UI 视觉规范和 Design Token，使后续页面和组件改造有统一的颜色、背景、文字、边框、圆角、阴影、间距、动效和状态标准。

## 当前情况

- `ui/src/index.css` 已通过 Tailwind 4 的 `@theme` 定义部分变量，包括 `--color-bg`、`--color-surface`、`--color-fg`、`--color-accent` 等。
- `ui/src/App.tsx` 会根据 `useThemeStore` 写入运行时 CSS 变量，保留暗色/亮色和强调色能力。
- `ui/src/store/useThemeStore.ts` 默认主题为暗色，默认强调色为 `#5199FF`。
- 当前业务组件大量直接写 `text-white/45`、`border-white/10`、`bg-white/[0.03]`、`rounded-xl` 等视觉值。
- 参考图呈现深色桌面软件风格：偏蓝黑背景、固定侧栏、蓝色强调、克制边框、表单控件高密度、分组标题带蓝色竖线。

## 实施方案

先审计当前 `index.css`、`App.tsx` 和基础 UI 组件中使用的视觉值，再在 `index.css` 中补充命名清晰的 CSS Variables。暗色主题按参考图作为主方向；亮色主题只保证变量存在和基本可用，是否同等精修待确认。

## 执行步骤

1. 搜索 `ui/src` 中散落的颜色、透明度、圆角、阴影、间距和状态色写法，归类为背景、文字、边框、控件、浮层、状态、布局。
2. 对照参考图和参考 HTML，确定暗色主题的基础色板：应用背景、侧栏背景、页面背景、控件背景、浮层背景、卡片/区块背景、主强调色、弱强调背景。
3. 在 `ui/src/index.css` 中补充或重命名设计变量，至少覆盖背景、文字、边框、品牌色、状态色、半径、阴影、间距、动效、z-index。
4. 调整 `ui/src/App.tsx` 中主题变量写入逻辑，使运行时变量与 `index.css` 的 token 语义一致。
5. 只在必要时更新 `ui/src/store/useThemeStore.ts` 的默认色值或派生色函数，保持持久化 key 和数据结构不变。
6. 运行 `npx tsc --noEmit`，确认类型不受影响。
7. 更新本任务执行记录和 `00-任务总览.md`。

## 涉及内容

- 待确认：如 token 过多，可新增 `ui/src/styles/` 下的样式分层文件，但默认优先只维护 `ui/src/index.css`。

### 修改文件

- `ui/src/index.css`：扩展和规范化 Design Token。
- `ui/src/App.tsx`：同步运行时主题变量写入。

# 桌面端 UI 系统化改造

## 任务目标

本任务用于在现有「说吧！」桌面端应用基础上，按参考设计图系统性完善 UI。目标是保留当前功能、业务逻辑、Tauri 通信、状态管理和桌面交互流程，逐步建立统一、专业、可持续维护的桌面端 UI 系统。

本次任务包含主窗口应用壳层、Design Token、基础 UI 组件、语音输入、实时字幕、录音识别、文稿对齐、设置页及复杂控件的视觉和布局改造。不包含后端识别能力重写、业务流程重做、引入大型 UI 框架、发布部署、替换现有技术栈，也不处理 `docs/task/录音识别与文稿对齐`。

## 重要说明

- 参考设计图优先于 `docs/others/stitch_speak_bar_ui_reconstruction` 中的参考 HTML；参考 HTML 只用于理解布局关系和局部样式。
- UI 改造不得破坏现有功能、Zustand store、Tauri invoke/event、业务 controller 和桌面交互流程。
- 不引入大型 UI 框架；优先使用 Tailwind CSS 4、CSS Variables 和现有组件体系。
- 涉及鼠标拖放、点击、悬浮、滑块、颜色选择、字幕预览等交互时，代码侧只提供验证步骤，最终由用户人工操作验证。
- 后续任务执行前必须阅读本文件、`01-实施方案.md` 和对应任务文件；完成后同步任务状态与执行记录。

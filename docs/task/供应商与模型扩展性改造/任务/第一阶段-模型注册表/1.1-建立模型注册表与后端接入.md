# 1.1 建立模型注册表与后端接入

## 任务状态

- 状态：已完成
- 所属阶段：第一阶段-模型注册表
- 前置任务：无
- 关联任务：1.2（前端消费同一份注册表）、2.1（连接器按注册表协议族分发）
- 是否可以并行：本任务是第一个任务，无并行对象

## 任务目标

建立一份结构化的 ASR 模型注册表作为唯一数据源，后端所有"按模型名判断能力/协议"的散落逻辑改为查表（保留前缀匹配作为表外模型的兜底），行为与现状完全一致。

## 当前情况

后端按模型名判断的位置（全量清单）：

| 位置 | 现有判断 | 对应注册表字段 |
|---|---|---|
| `src-tauri/src/providers/alibabacloud/protocol.rs:70-76` `realtime_asr_family()` | `starts_with("qwen3-asr-flash-realtime")` → QwenRealtime，否则 DashscopeDuplex | 协议族 |
| `src-tauri/src/providers/alibabacloud/protocol.rs:87` | `starts_with("fun-asr") || starts_with("paraformer")` 才带 `vocabulary_id` | 支持热词 |
| `src-tauri/src/providers/alibabacloud/transcription.rs:511-517` `upload_for_model()` | 按模型名分 filetrans / 短音频 / fun-asr-flash / 其余（OSS）4 路 | 文件转写通路 |
| `src-tauri/src/commands/transcription.rs` `uses_async_transcription_task()` | 判断异步 OSS 还是同步短音频 | 文件转写通路 |
| `src-tauri/src/providers/alibabacloud/customization.rs:11-15` `VOCABULARY_TARGETS` | 热词目标模型硬表（fun-asr-realtime-2026-02-28 / fun-asr-realtime / fun-asr + 各自前缀） | 热词目标（前缀是热词系统私有数据，仍留在此表） |
| `src-tauri/src/providers/alibabacloud/protocol.rs:47` `DEFAULT_REALTIME_MODEL`、`transcription.rs:12` `DEFAULT_TRANSCRIPTION_MODEL` | 兜底默认模型 | 默认标记 |

模型全集（现状 9 个）：`fun-asr-realtime-2026-02-28`、`fun-asr-realtime`、`qwen3-asr-flash-realtime-2026-02-10`、`qwen3-asr-flash-realtime`（实时）；`fun-asr-flash-2026-06-15`、`fun-asr`、`qwen3-asr-flash-filetrans`、`qwen3-asr-flash`、`qwen3-asr-flash-2026-02-10`（文件）。各自的场景可用性见 `ui/src/features/asr/modelOptions.ts` 三张清单（含注释里的原因，如 qwen3-asr-flash 无时间戳不能进录音识别下拉）。

可复用：`RealtimeAsrFamily` 枚举可保留为注册表"协议族"字段的 Rust 表示。

## 实施方案

按重要记录 001 推荐方案：在仓库根新建 `shared/asr-models.json`，每条记录含 `id`、`label`、`providerKind`、`category`（realtime/file）、`protocol`（dashscope-duplex / qwen-realtime / file-async-oss / file-sync-funasr-flash / file-sync-qwen）、`supportsVocabulary`、`supportsAlignmentTimestamps`、`scenes`（dictationRealtime / dictationFile / subtitles / transcription 数组）、`isDefaultRealtime` / `isDefaultFile`。

Rust 侧新建 `src-tauri/src/providers/registry.rs`：`include_str!("../../../shared/asr-models.json")` + serde 解析 + `once_cell::sync::Lazy` 静态表；提供 `model_info(id) -> Option<&ModelInfo>`、`realtime_family(id)`（查表，miss 时按现有前缀规则兜底）、`supports_vocabulary(id)`、`file_route(id)` 等查询函数。

本任务只改后端消费点；`modelOptions.ts` 与注册表的一致性由 1.2 处理，期间两份数据并存（内容一致，以现状为准填写 JSON）。

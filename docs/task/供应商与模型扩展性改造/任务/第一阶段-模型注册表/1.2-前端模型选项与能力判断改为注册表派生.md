# 1.2 前端模型选项与能力判断改为注册表派生

## 任务状态

- 前置任务：1.1

- 是否可以并行：可与 2.1 并行（互不触碰对方文件）

## 任务目标

`ui/src/features/asr/modelOptions.ts` 不再手工维护三张下拉清单和 9 个布尔函数，全部从 `shared/asr-models.json` 派生；对外导出的函数名与签名尽量保持不变，使 10 个消费文件无需改动或仅微调。

## 当前情况

- `ui/src/features/asr/modelOptions.ts` 手工维护：
  - `REALTIME_ASR_MODEL_OPTIONS`（4 项）、`FILE_ASR_MODEL_OPTIONS`（3 项）、`DICTATION_FILE_ASR_MODEL_OPTIONS`（5 项）、`DICTATION_ASR_MODEL_OPTIONS`（前两者拼接）；
  - `DEFAULT_REALTIME_ASR_MODEL` / `DEFAULT_FILE_ASR_MODEL` 常量；
  - `isSupportedRealtimeModel` / `isSupportedFileModel` / `isSupportedDictationModel` / `isDictationFileModel` / `isQwenRealtimeModel` / `isQwenFileModel` / `isQwenShortAudioFileModel` / `isFunAsrFlashFileModel` / `supportsAlignmentTimestamps`。
- 消费方（10 个文件）：`views/RealtimeSubtitlesPanel.tsx`、`views/DictationShortcutsPanel.tsx`、`views/TranscriptionView.tsx`、`features/dictation/controller.ts`、`features/transcription/controller.ts`、`features/compare/models.ts`、`store/useDictPrefs.ts`、`store/useTranscriptionStore.ts`、`store/useSubtitleStore.ts`。
- 注意：`isQwen*` / `isFunAsrFlashFileModel` 这类"协议族/特定模型"判断，语义上应改为按注册表能力字段判断（消费方真正关心的是"有没有时间戳""走哪条通路"），执行时逐个消费点确认真实意图再决定是替换成能力函数还是保留同名封装。
- Vite root 是 `ui/`，import 仓库根的 `shared/asr-models.json` 需要相对路径 `../../shared/`（JSON import Vite 原生支持，会内联进 bundle）；若路径解析有问题，可在 `vite.config.ts` 加 alias（改动需最小）。

## 实施方案

新建 `ui/src/features/asr/modelRegistry.ts`：import JSON、定义 TS 类型（与 Rust 侧 `ModelInfo` 字段对齐）、提供按场景过滤下拉清单和能力查询的函数。`modelOptions.ts` 改为从 registry 派生并 re-export 原有名称（保持消费方零改动）；确认无引用残留后，把纯前缀判断类函数的实现改为查表 + 前缀兜底（与 Rust 侧兜底规则一致）。

## 执行步骤

1. 新建 `ui/src/features/asr/modelRegistry.ts`：JSON import、类型定义、`optionsForScene(scene)`、`modelInfo(id)`、`supportsAlignmentTimestamps(id)` 等查询函数，表外模型名按前缀兜底（规则与 `registry.rs` 一致）。

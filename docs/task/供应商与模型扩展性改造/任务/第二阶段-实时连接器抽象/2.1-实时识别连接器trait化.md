## 任务目标

把实时识别的协议族差异（DashScope duplex / Qwen realtime）收敛为一个 `RealtimeAsrConnector` trait，`commands/asr.rs` 的会话循环与静音测试不再出现 `match family`；新增协议族 = 在供应商模块内新增一个 trait 实现 + 注册表登记，会话循环零改动。**纯重构，行为不变。**

## 当前情况

- 协议差异现分布在 5 处 `match`：
  - `commands/asr.rs:41-63`：连接请求（`funasr_ws_request` vs `qwen_realtime_request`，见 `providers/alibabacloud/urls.rs`）与首消息（`build_run_task_message`+task_id vs `build_qwen_session_update_message`）；
  - `commands/asr.rs:343-355` `send_audio_message`：Binary vs base64 JSON；
  - `commands/asr.rs:357-372` `send_finish`：`build_finish_task_message(task_id)` vs `build_qwen_finish_message()`；
  - `providers/alibabacloud/protocol.rs:180-185` `parse_server_message`：按 family 分发到两个解析函数；
  - `commands/asr.rs:477-553` `run_asr_silence_test`：写死 duplex 协议并拒绝 Qwen 模型。
- 会话循环消费的抽象事件 `FunAsrEvent`（`protocol.rs:49-56`）已协议无关：Started / Partial(String) / Final(String) / TaskFinished / TaskFailed / Other(Value)。
- 连接器需要携带的状态：`task_id`（仅 duplex）、`FunAsrParams`、`model`。`FunAsrProtocol` 结构体（`asr.rs:8-14`）中的 `started` / `pending`（task-started 前的音频缓冲）是**会话循环的状态**而非协议状态，留在循环里。
- 可复用：两个协议的全部消息构造/解析函数已在 `protocol.rs`，搬运即可，不需要重写。

## 实施方案

在 `providers/mod.rs`（或新文件 `providers/connector.rs`）定义：

```rust
pub enum AsrEvent { Started, Partial(String), Final(String), TaskFinished, TaskFailed { code: String, message: String }, Other(Value) }

pub trait RealtimeAsrConnector: Send {
    fn connect_request(&self) -> Result<Request<()>, String>;
    fn start_messages(&self) -> Vec<Message>;
    fn audio_message(&self, bytes: Vec<u8>) -> Message;
    fn finish_message(&self) -> Message;
    fn parse_message(&self, text: &str) -> AsrEvent;
}
```

`providers/alibabacloud/` 内实现 `DashscopeDuplexConnector { api_key, task_id, params, model }` 与 `QwenRealtimeConnector { api_key, params, model }`；供应商模块暴露工厂 `alibabacloud::realtime_connector(profile_config, model) -> Result<Box<dyn RealtimeAsrConnector>, String>`（内部按 registry 协议族选实现）。`FunAsrEvent` 更名 `AsrEvent`（波及 `prelude.rs`、`asr.rs`、`protocol.rs`，纯改名）。

`run_asr_silence_test` 同步改用连接器；其"仅支持 duplex"的限制维持现状（Qwen 依旧返回明确错误，错误判断依据从模型前缀改为注册表协议族）。
